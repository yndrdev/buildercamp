## Decisions
- **Claude Haiku 4.5 for chat, Haiku for analysis** — fast + cheap for conversation turns. Could upgrade analysis to Sonnet for better quality.
- **Deepgram over Whisper** — lower latency, generous free tier, simple REST API
- **PIN auth over proper auth** — quick solution for now. User may want Clerk or proper auth later.
- **Questions trimmed to 5 per session** — user explicitly requested 5-10 max for busy professionals
- **Conversations table separate from submissions** — backward compatible, old form data still accessible
- **shadcnuikit registry** — components pulled via CLI with bearer token, not downloaded files

## Supabase Tables
- `clients` — multi-tenant, keyed by slug
- `session_groups` — per client, ordered
- `questions` — per session group (5 per session after trim)
- `client_roles` — per client role definitions
- `client_knowledge` — per-client AI context (role_context, client_context, session_insight)
- `submissions` — old form submissions
- `conversations` — new chat conversations (messages JSONB, analysis JSONB, resume_token)

## API Routes
- `POST /api/chat/start` — init conversation, return greeting + context
- `POST /api/chat` — streaming chat with Claude
- `PATCH /api/chat` — update conversation metadata
- `POST /api/transcribe` — Deepgram voice transcription
- `POST /api/analyze` — post-completion transcript analysis
- `POST /api/acknowledge` — legacy, can be removed
